Global    | .data section      | 0x404018           | operation_count, fixed address, full program lifetime

PLT/GOT Usage

- PLT (Procedure Linkage Table): Contains stubs for library functions (malloc@plt, printf@plt, free@plt)
- GOT (Global Offset Table): Stores actual addresses of functions after resolution
- First call: PLT triggers dynamic linker to resolve the function
- Later calls: Direct jump to real libc function (lazy binding)

Key Insights

1. _start is the real ELF entry point. It calls __libc_start_main, which sets up the C runtime and then calls main().
2. Memory is used in three regions: stack for locals, heap for dynamic array, .data for globals.
3. System calls are triggered by library functions: brk (malloc), write (printf), exit_group (exit).
4. Dynamic linking uses PLT/GOT mechanism with lazy binding for efficiency.
5. High-level C constructs (loops, if-statements, malloc/printf) translate to low-level assembly and syscalls.
